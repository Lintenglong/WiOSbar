# 多灵动岛显示策略设计

## 目标

在主设置面板增加“显示策略”：默认“只显示最新”，可切换为“同时显示多个”。同时显示时，新非时钟状态追加到已有状态右侧，旧状态不被覆盖；时钟状态永远不进入多岛列表，继续作为单一空闲/常驻状态覆盖显示。

## 交互规则

- `LatestOnly`：保持现有单灵动岛行为。
- `Multiple`：非时钟事件会保留已有非时钟状态，并把最新事件放到最右侧。
- 时钟事件永远清空/忽略多岛堆叠，不产生多个时间岛。
- 居中位置按整组总宽度居中；左侧锚点保持第一个岛贴近原基准点；右侧锚点让整组右边缘贴近原基准点。
- 最新岛仍使用完整内容、悬停卡片和现有弹簧动画；旧岛以紧凑快照显示，避免旧状态被新状态覆盖。

## 架构

新增纯逻辑模型负责策略和布局：`IslandStackPolicy` 决定事件是否追加，`IslandGroupLayout` 根据锚点、尺寸和间距计算整组窗口尺寸与每个槽位偏移。UI 层消费这个模型，主窗口仍保持单个最新岛的完整动画，旧岛使用轻量快照窗口渲染，避免把现有主岛动画拆进多份命名控件后重新引入卡顿。

## 测试

测试覆盖：默认策略、同时显示追加、时钟排除、居中/左上/右上定位、最大保留数量、切换回单岛时清空旧岛。构建仍以 `dotnet run --project FluidBar.Tests\FluidBar.Tests.csproj` 和 `dotnet build -c CodexVerify` 验证。
