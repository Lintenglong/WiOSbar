# Security Policy

## 🔒 报告安全漏洞

我们非常重视 FluidBar 的安全。如果您发现了安全漏洞，请**不要**公开报告，而是通过以下方式私下联系我们：

### 报告方式

**Email**: security@example.com

**内容应包括**：
- 漏洞描述
- 复现步骤
- 潜在影响
- 可能的修复建议（可选）

### 响应时间

- **确认收到**：24 小时内
- **初步评估**：72 小时内
- **修复发布**：根据严重程度 1-30 天

## 🔐 安全最佳实践

### 用户建议

1. **从官方来源下载**
   - GitHub Releases 页面
   - 验证文件哈希值

2. **保持更新**
   - 定期检查新版本
   - 及时安装安全更新

3. **谨慎配置**
   - 不要在配置文件中存储敏感信息
   - 天气 API Key 应妥善保管

4. **日志审查**
   - 定期检查 `%AppData%\FluidBar\logs\`
   - 及时报告异常行为

### 开发者建议

1. **依赖管理**
   - 定期更新 NuGet 包
   - 仅使用官方包

2. **代码审查**
   - 所有 PR 需经过审查
   - 关注输入验证

3. **数据处理**
   - 不要记录敏感信息到日志
   - 正确处理用户输入
